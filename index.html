About

A multidisciplinary linguist & developer with a strong passion for anything linguistics, namely: syntax, theoretical linguistics and computational linguistics. Patrick grew up in Southwest Florida where he spent his spare time feeding his passion for self-learning. Building motorcycles and installing car stereo systems in high school turned into a job with the U.S. Army working on power generation equipment systems. While in the service, Patrick became intrigued by the power of language on a tour of duty in Iraq in 2011 where he spoke to an Iraqi Translator. Shortly after his contract ended, Patrick used his G.I. Bill to go to school and pursue his new-found passion. Language.

He received his B.A. In Linguistics from Stony Brook University in New York. While studying, his main interests were both theoretical and computational linguistics. As a part of his program, he studied one year of both French and Arabic; studying Arabic with the government STARTALK program receiving an ACETFL novice-mid proficiency certificate.

While at Stony Brook, he gained experience in developing while conducting research under the State University Of New York Research Foundation where he carried out a phonological experiment which included coding in Java, HTML/CSS and JavaScript. Also, in the department's "Special Topics in Linguistics: Computational Linguistics" course, he programmed extensively in python and focused on natural language processing.

Patrick got his first taste of linguistic experience while working for a year at Stony Brook's Multilingual and Multicultrual Communications Center (MIC) as a Linguist/Student assistant. While at the MIC, he worked with top global scholars to examine communicative adapatations by individuals as they transcend language and cultures. His time here was spent doing reaserch, conducting interviews and generating knowledge on the impact of global mobility and technologial innovation on language practices.

Now he writes computer programs to both analyze natural language and solve linguistic problems.

Java!

    1.) When would you use '=' vs '==' vs '.equals'?

    Roughly, we can describe ‘=‘ as an assignment, ‘==‘ as an operator and ‘.equals()’ as a method.

      ’=‘ is used to assign a value to a variable. In Java programming, you must first declare a variable before you assign a method to it. Assigning a value to a variable as also known as initializing the variable. The syntax for declaring a variable comes with it’s data type (int) followed by the variable name (x).

      …

      -> int x;

      Assigning a value can be done on the same line as declaring a variable. This method can be used for saving space, and sometimes more readability, in a program by declaring and assigning a value in the beginning of the program.

      …

      -> int x = 1

      ‘==‘ is an operator that compares the value of two objects to see if they refer to the same string instance. It used to compare whether two operands are equal or not. You can use this operator to compare primitive types (int, char, bool) or object references (String string1 = new String( “Value” ). If the value of both primitive instance’s are equal, the result of the boolean (true or false expression) would be “true”:

      …

      int a = 8;

      int b = 9-1;

      int c = 4;

      ( a == b ) // True

      ( a == c ) // False

      ‘.equals().’ is a method that compares the “value” inside a string instance. This method does not compare about object reference and only compares values of the instances. The only comparison made is that the array of characters are in the same order. This method also returns a boolean value as a result, returning “true” if the characters are in the same order and “false” if otherwise.

      …

      String string1 = “Patrick Gaston”;

      String string2 = “Pat Gaston”;

      String string3 = “Patrick Gaston”;

      string1.equals(string2); // False

      string1.equals(string3); // True

    2.) What does it mean to 'cast a double to an int'?

    We can think of ‘cast’ as being synonymous with ‘push’ in this instance. When we say we are ‘casting a double to an int’, essentially we are taking the value stored in double (a decimal value; i.e. 3.05) and storing it in an int (whole number; i.e. 3). In this process, the decimal is implicitly dropped and rounded down regardless of decimal value. Casting a double to an int can be done in multiple ways:

      Casting in one step:

      …

      int x = (int)(11.26); // This will return: x = 11

      casting in 2 steps:

      …

      double b = 3.4;

      double c = 3.9;

      int x = (int)(b); // This will return: x = 3

      int y = (int)(c); // This will ALSO return: x = 3

    3.) What happens when you include 'import java.util.Scanner;' in your code? Where does this functionality come from? What is the 'java.' in 'java.util.Scanner;?

    To get a better understand the function of import java.util.Scanner; we need to understand what a package is. A package is a bin for classes that we can call to use in our programs when we need to. This command is based on hierarchy (think of opening files in your computer and the file path being ‘Java/Util/Scanner’).

      import
      To call upon them, we need to first tell our program that we are seeking a package. This can be done with import

      java
      After we tell our program we need to import, we then need to tell it where to look. java tells our program to look in the java file for…

      util
      Util is short for utility and contains a certain type of class we are looking for (utility)

      Scanner
      finally, we tell the program what class we are looking for. Here, we call for the Scanner class. This class is predefined in the system for taking inputs from the user (in the form of keyboard strokes).

    4.) What is 'void' in a method declaration? How does this differ from declaring a method with a return type?

    Declaring a method can help us do many things in a program. For instance, we use methods already created in Java when we use predefined methods like pause and getHeight to save us from typing these methods all the time. The thing about methods is that they keep going until they hit a return value. Essentially, the method calculates (or does whatever you tell it to) until it hits return, then it returns whatever you specify outside of the method to facilitate your program.

      so why void? The thing is, we don’t always want our method to return a value. Putting return void serves 2 purposes. The return tells our method to stop running and the void tells our method to return no value

      ...

      return void;

    5.) When it comes to methods, what is the difference between a "parameter" and an "argument"? Is there one? When would you use one versus the other?

    When I first started programming, I would use “parameter” and “argument” interchangeably. It wasn’t after a while that I realized they were two separate entities and clearly distinguishable once you took a closer look. Let’s revisit the ‘.equals()’ method from earlier.

      A parameter is a value that is set in the method definition when it is created. When .equals() was made, they created parameters in the form of a value within the parentheses. Roughly, the ‘()’’ in the method translates to “the value that is in this set of parentheses.” So, we can read it as follows:

      “The [method] named “.equals” will conduct the following algorithm (calculations, comparison, analysis, etc.) on the [value] within these parentheses.”

      An argument is what is passed through the method after it is defined. So if we use the ‘.equals()’ method:

      …

      String string1 = “Patrick Gaston”;

       

      String string2 = “Pat Gaston”;

      in the following piece of code, “string2” is the argument that was pass through the method.

      …

      string1.equals(string2); // False
